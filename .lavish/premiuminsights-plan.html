Risks Things to watch

Roll detection heuristic sensitivity

The 1-day window may produce false positives if two unrelated trades happen to be a BTC + STO on the same underlying on the same day. The window_days parameter lets you tune this without changing the call sites.

Journal P&L reveal UX (session state)

The "hide P&L until rated" behavior relies on Streamlit session state. A page refresh clears the hidden state. The rating is persisted in DB, but if the user refreshes mid-review the P&L may become visible before rating is saved — acceptable for a single-user personal tool.

profit_factor when no losses

Σwins / |Σlosses| is undefined when there are 0 losing trades. Return float('inf') or a sentinel, and render as "∞" or "–" in the UI so the metric card doesn't display nan.

Turso cloud sync of journal rows

The local-to-cloud sync tool (tools/sync_to_cloud.py) currently syncs deals and wheels. If you use Turso in cloud mode, extend the sync to include the journal table — otherwise journal entries won't survive a local DB wipe.
